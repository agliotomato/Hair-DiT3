## 오늘 한 일

### 1. 색 following 원인 분석 및 실험 설계

**문제**: hair-dit3에서 sketch stroke 색을 따르지 않음.

**배제된 원인들 (실험으로 확인):**
- Compositor train-inference mismatch → OFF 테스트 → 색 안 따름 → 원인 아님
- Matte-gated blending → 제거 패치 → 색 안 따름 (품질은 향상) → 원인 아님
- pos_embed_input zero-init → diffusers가 두 모델 동일하게 zero_module → 원인 아님

**확인된 근본 원인:**
- `L_bg (weight=3.0)`이 `L_flow (1.0)`을 압도 → 모델이 sketch 색 신호 무시
- 명시적 색 대응 loss 없음 → frozen SD3.5 prior의 자연 헤어색이 GT와 우연히 일치

### 2. 실험 브랜치 `exp/color-loss` 생성 및 구현

**변경 내용:**

| 파일 | 변경 사항 |
|---|---|
| `src/losses/hair_s2i_loss.py` | `L_color` 추가: `MSE(pred_z0 * matte, sketch_latent * matte)` |
| `src/models/hair_s2i_net.py` | `forward()` → `(noise_pred, noise, sketch_latent)` 3-tuple 반환 |
| `src/training/trainer.py` | sketch_latent 언패킹, criterion 전달, seed 고정 코드 추가 |
| `src/training/augmentation.py` | `StrokeColorSampler p=0.0 → p=1.0` 복원 |
| `configs/base.yaml` | `bg: 3.0→0.5`, `color: 1.5` 추가, `seed: 42` 추가 |
| `configs/phase2.yaml` | 동일 + `wandb_run_name: phase2_color_loss_bg0.5_c1.5_seed42` |

**L_color 설계 원리:**
```
sketch_latent [B,16,64,64] — VAE encode(sketch), 색 정보 포함
pred_z0       [B,16,64,64] — 예측 clean latent
L_color = MSE(pred_z0 * matte, sketch_latent * matte)
→ 헤어 영역에서 pred_z0가 sketch_latent를 따르도록 강제
→ VAE decode 없이 latent space에서 직접 계산 (VRAM 절약)
```

**실험 조건 (2안 채택):**
```
L_flow   1.0
L_bg     0.5  (3.0 → 0.5: 색 신호가 묻히지 않도록)
L_color  1.5  (신규: L_flow보다 높게 → 색 following 우선)
L_lpips  0.2
L_edge   0.15
```

**재현 규칙:**
- 모든 실험 config+seed 기록 → `configs/phase2.yaml`에 `seed: 42` 명시
- 3회 seed 변경 후 mean ± std 보고 (seed: 42, 123, 777)

### 3. 평가 스크립트 구축 (`scripts/evaluate.py`)

실험계획서 크리티컬 패스: **평가 스크립트 → Ours inference → 정량 계산 → Figure → 논문**

**구현된 지표 전체:**

| 지표 | 영역 | 설명 |
|---|---|---|
| `psnr_bg` | 배경 | 배경 보존 PSNR |
| `ssim_bg` | 배경 | 배경 보존 SSIM |
| `ssim_hair` | 헤어 | GT 대비 헤어 영역 SSIM (per-pixel map 평균) |
| `lpips_hair` | 헤어 | GT 대비 헤어 영역 LPIPS (VGG) |
| `boundary_ssim` | 경계 | dilate XOR erode strip SSIM |
| `boundary_lpips` | 경계 | 경계 strip LPIPS |
| `edge_iou` | 구조 | Canny(generated) ∩ sketch / union |
| `chamfer_dist` | 구조 | edge ↔ sketch 양방향 최소 거리 평균 (↓ 좋음) |
| `fid_hair` | 분포 | InceptionV3 pool3 → Fréchet (hair crop, 루프 후 계산) |
| `fid_boundary` | 분포 | 경계 strip crop FID |
| `arcface_cos` | identity | ArcFace cosine (insightface 필요, 현재 nan) |

**사용법:**
```bash
python scripts/evaluate.py \
  --pred_dir   results/batch_infer_final2 \
  --gt_dir     dataset3/braid/img/test \
  --matte_dir  dataset3/braid/matte/test \
  --sketch_dir dataset3/braid/sketch/test \
  --method_name ours \
  --output eval_results_ours.csv
  # --no_fid  : FID 생략 (빠른 디버그)
```
