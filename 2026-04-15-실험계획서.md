## 0. 논문의 핵심 주장 (실험으로 증명해야 할 것)

| # | Claim | 핵심 실험 |
|---|---|---|
| C1 | Matte-gated ControlNet이 sketch에 충실한 hair를 생성한다 | SOTA 비교 (§2) |
| C2 | Hair structure는 배경에 독립적이고, appearance는 DiT attention으로 자연스럽게 조화된다 | 배경 독립성/조화 분석 (§3) |
| C3 | 같은 모델이 generation-only / inpainting 두 모드를 지원한다 | 모드 비교 (§3) |
| C4 | Matte-gated blending의 각 설계 요소가 성능에 기여한다 | Ablation (§4) |

## 1. 비교군 설정

### 검색 규칙 준수 (CLAUDE.md 2026-04-15 업데이트)

> 2025~2026년 최신 연구 우선, 전체 비교군의 30% 이상이 2025~2026년 논문

### 비교군 선정 원칙

스케치 기반 비교가 핵심입니다. 다만 **네 가지 계층**으로 구성합니다:

| 계층 | 목적 | 비교 방식 |
|---|---|---|
| **Tier 1: Sketch/Stroke 기반 hair editing** | 동일 task에서의 직접 비교 | 동일 sketch+matte 입력 |
| **Tier 2: Hair 특화 diffusion (2024-2025)** | hair 도메인 내 최신 방법 비교 | 각 방법의 최적 입력 사용 |
| **Tier 3: General inpainting (2024-2025)** | "hair 전용 설계가 왜 필요한가" 증명 | 동일 mask, sketch를 condition으로 |
| **Tier 4: Reference 기반 (1개)** | paradigm 비교 — intro motivation | qualitative만 |

### 2025-2026 서베이 핵심 발견

| 발견 | 의미 |
|---|---|
| **2025-2026에 sketch 기반 hair editing 신규 논문 없음** | Hair-DiT2의 독자적 포지셔닝이 매우 강함 |
| **DiT 기반 inpainting 급증** (DiTPainter, EraserDiT) | DiT 아키텍처 선택이 트렌드 선도 |
| **Hair 도메인이 Video/3D로 확장** | 2D image editing 자체가 "해결된 문제"가 아님을 확인 |
| **Stable-Hair이 AAAI 2025에 정식 발표** | Reference 기반의 최신 대표작 |

### Tier 1: Sketch/Stroke 기반 Hair Editing (핵심 비교군)

#### 1-1. SketchHairSalon ★필수

| 항목 | 상세 |
|---|---|
| 출처 | SIGGRAPH Asia 2021 (ACM TOG) |
| 입력 | Hair sketch (흑백 선화) + hair mask |
| 출력 | Hair region 생성 (composite 후처리 필요) |
| 코드 | https://github.com/chufengxiao/SketchHairSalon |
| 비교 적합성 | **최고** — 유일한 동일 task, sketch 기반 |
| 공정성 이슈 | (a) 흑백 sketch → Hair-DiT2는 colored sketch. 공정 비교를 위해 흑백 변환 실험 필요. (b) GAN 기반 → 아키텍처 세대 차이 (이건 우리의 장점으로 포장) |
| 비교 포인트 | Sketch fidelity, boundary quality, composite artifact 유무 |
| 연도 참고 | 2021년이지만 **이 분야의 유일한 직접 비교 대상**이므로 필수 포함 |

#### 1-2. HairDiffusion ★강력 권장

| 항목 | 상세 |
|---|---|
| 출처 | arXiv 2024.10 |
| 입력 | Hair stroke maps + color strokes |
| 출력 | Full image editing |
| 기반 모델 | SD1.5 기반 latent diffusion |
| 비교 적합성 | **매우 높음** — colored stroke/sketch 기반, diffusion 모델 |
| 공정성 이슈 | SD1.5 vs Hair-DiT2의 SD3.5 (아키텍처 세대 차이). Stroke map 형식 변환 필요할 수 있음 |
| 비교 포인트 | **가장 공정한 비교 대상** — 입력 형식이 거의 동일 |

## 2. 실험 A: SOTA 정량 비교 (Main Results)

### 2.1 데이터셋

| 데이터셋 | 용도 | 규모 |
|---|---|---|
| **Hair-DiT2 test set** (braid/unbraid) | 주 평가 | test split 전체 |
| **SketchHairSalon dataset** | 공정 비교 (상대편 데이터) | test split |
| **CelebA-HQ subset** | 범용 평가 (sketch 자동 추출) | 500장 |

### 2.2 입력 통일 프로토콜

```
모든 baseline에 동일한 입력 제공:

1. Hair sketch: Hair-DiT2 학습 데이터의 colored sketch
   - SketchHairSalon용: grayscale 변환
   - HairCLIPv2용: 해당 포맷으로 변환
   - BrushNet/FLUX용: ControlNet condition 또는 참조

2. Hair matte: 동일한 soft alpha matte
   - binary mask 필요 시: threshold 0.5

3. Background: 동일한 원본 이미지 (hair 영역 masked)

4. Resolution: 512×512 통일
```

### 2.3 평가 지표

#### 구조 충실도 (Sketch Fidelity)

| 지표 | 설명 | 계산 방법 |
|---|---|---|
| **Edge IoU** | 생성 hair의 edge와 input sketch의 일치도 | Canny(generated) ∩ Canny(sketch) / union, hair region 내 |
| **Chamfer Distance** | Edge 간 거리 | 생성 edge ↔ sketch edge 양방향 최소 거리 평균 |
| **Sketch LPIPS** | Sketch와 생성 결과의 perceptual 유사도 | LPIPS(edge_map(generated), sketch), hair region |

#### 생성 품질 (Generation Quality)

| 지표 | 설명 | 계산 방법 |
|---|---|---|
| **FID** | 생성 분포와 실제 분포의 거리 | Hair region crop → FID (최소 500장) |
| **LPIPS (vs GT)** | GT 대비 perceptual 유사도 | Hair region masked LPIPS |
| **SSIM (vs GT)** | GT 대비 구조적 유사도 | Hair region masked SSIM |
| **PSNR (vs GT)** | GT 대비 pixel 유사도 | Hair region masked PSNR |

#### 경계 품질 (Boundary Quality)

| 지표 | 설명 | 계산 방법 |
|---|---|---|
| **Boundary FID** | 경계 영역 자연스러움 | matte 0.1~0.9 영역 strip crop → FID |
| **Boundary LPIPS** | 경계 영역 GT 대비 유사도 | 경계 strip LPIPS |

#### Identity 보존

| 지표 | 설명 | 계산 방법 |
|---|---|---|
| **Face LPIPS** | 얼굴 영역 보존도 | (1-matte) 영역 LPIPS vs GT |
| **ArcFace cosine** | 얼굴 identity 보존 | ArcFace embedding cosine similarity |

## 3. 실험 B: 배경 독립성 & Appearance 조화 분석

### 3.1 Structure Independence 검증

**목적**: 배경이 바뀌어도 hair의 **구조(strand pattern, shape)**가 동일하게 유지됨을 정량 증명

```
설계:
  고정: sketch, matte, seed
  변수: background 5종
    (a) 없음 (Mode A, black/noise)
    (b) white
    (c) original GT background
    (d) random person background (다른 인물)
    (e) scene background (풍경, 인물 없음)
  
  샘플: N ≥ 50장

측정 (모든 pair 조합):
  - Hair region SSIM(a↔b), SSIM(a↔c), ..., SSIM(d↔e)
  - Hair edge map IoU (Canny edge 비교)
  - Strand direction consistency (orientation histogram 비교)

결과 형식:
  Table 2: Pairwise hair structure similarity across background conditions
  
  Pair        | SSIM↑  | Edge IoU↑ | OrientHist↑
  a↔b (none↔white)  | 0.xx | 0.xx | 0.xx
  a↔c (none↔orig)   | 0.xx | 0.xx | 0.xx
  a↔d (none↔random)  | 0.xx | 0.xx | 0.xx
  ...

기대: 모든 pair에서 높은 유사도 → "structure is background-independent"
```

### 3.2 Appearance Harmonization 검증

**목적**: hair의 **외관(조명, 색온도)**이 배경에 맞게 적절히 조화됨을 증명

```
설계:
  고정: sketch, matte, seed
  변수: 극단적 조명 조건 4종
    (a) warm light (석양, 따뜻한 톤)
    (b) cool light (형광등, 차가운 톤)
    (c) bright (밝은 실내)
    (d) dark/backlit (어두운/역광)

  각 조건의 background는 조명 편집(color grading) 또는 다른 이미지 사용

측정:
  - Hair region color histogram (Hue/Saturation 분포)
  - Hair region average color temperature (K)
  - Background color temperature와의 상관관계 (Pearson r)

결과 형식:
  Figure Y: Hair appearance adapts to scene lighting
  
  4열 비교: warm bg → warm-toned hair | cool bg → cool-toned hair | ...
  + Color temperature scatter plot (x: bg temp, y: hair temp) → 양의 상관 기대
  
기대:
  - Structure (edge) 동일 → C2a 확인
  - Color temperature에 양의 상관 → C2b 확인 ("DiT attention이 암묵적으로 조화")
  - 상관이 없으면 → limitation으로 기술하거나 harmonization 모듈 추가
```

### 3.3 Mode A vs Mode B 비교

## 4. 실험 C: Ablation Studies

### 4.1 Matte-Gated Blending Ratio

```
변수: blend_start = {0, 3, 6, 9, 12} (= 0%, 25%, 50%, 75%, 100%)

  - 0 (0%):   12개 블록 모두 matte gating → hair 영역에만 전부
  - 3 (25%):  앞 3 full + 뒤 9 gated
  - 6 (50%):  앞 6 full + 뒤 6 gated ← 현재
  - 9 (75%):  앞 9 full + 뒤 3 gated
  - 12 (100%): 12개 블록 모두 full → gating 없음

측정: Hair LPIPS, Edge IoU, Background PSNR (비헤어 영역 보존도)

⚠️ 재학습 필요 여부 확인:
  - Inference에서만 blend_start 변경 가능하면 → 빠름 (1일)
  - 재학습 필요하면 → 각 설정 재학습 (3-5일)
```

### 4.2 CFG Scale

```
변수: scale = {1.0, 3.0, 5.0, 7.5, 10.0, 15.0}

측정:
  - Sketch fidelity (Edge IoU) — scale↑ → fidelity↑ 기대
  - Naturalness (FID) — scale 너무 높으면 → 부자연스러움
  
결과: Trade-off curve (x: Edge IoU, y: FID)
  → "CFG scale로 sketch 충실도 vs 자연스러움을 사용자가 제어 가능"
```

### 4.3 Loss Function Ablation

```
변수:
  (a) L_flow only
  (b) + L_bg (배경 보존)
  (c) + L_lpips (perceptual)
  (d) + L_edge (sketch alignment) ← 현재

⚠️ 반드시 재학습 필요. 각 설정별 Phase 1+2 학습.
```

### 4.4 Compositor 유무

```
변수:
  (a) Compositor 사용 (현재)
  (b) Compositor 미사용 (순수 DiT 출력)

측정: Boundary quality, Background preservation, Hair quality
  → Compositor의 기여도 증명
```

### 4.5 Curriculum Learning 효과

```
변수:
  (a) Phase 1 (unbraid) only → braid 평가
  (b) Phase 2 (braid) directly (pretrain 없이)
  (c) Phase 1 → Phase 2 (현재)

측정: Braid structure quality, Loss curve 비교
  → Curriculum learning의 필요성 증명
```

## 6. 실험 E: User Study

### 설계

```
참가자: 20명 이상 (비전문가 + 그래픽 전문가 혼합)
샘플: 30~50 쌍

평가 항목 (각각 A/B forced-choice):
  Q1. Sketch Fidelity: "어떤 결과가 스케치를 더 정확히 따르나요?"
```
