# [0416] 서현택

## SketchHairSalon 동작 확인

### color_coding 동작 방식

- **학습 시**: `color_coding()` 함수로 sketch stroke 영역을 GT 이미지의 실제 픽셀 색으로 교체 (augmentation)
  - 67% 평균 색, 33% 랜덤 픽셀 샘플링
- **추론 시**: `color_coding()` 미사용. sketch의 non-zero 픽셀 색을 그대로 입력으로 사용

### 공정성 이슈 재검토

| 조건 | SketchHairSalon | Hair-DiT3 (EHC) |
|---|---|---|
| 입력 sketch | 흑백 or 컬러 (그대로 사용) | colored sketch |
| 색 반영 | 추론 시 sketch 색 그대로 입력 | stroke 색 → 생성 헤어에 반영 (목표) |

**결론:** 공정 비교를 위해 동일한 colored sketch를 양쪽 모두에 입력해야 함.
흑백 sketch를 SketchHairSalon에 넣으면 색 정보가 없으므로 불리한 조건 → 같은 colored sketch 사용이 올바른 비교 방식.

따라서 ours도 sketch 색을 따라가는 헤어 생성 버전이 필요함.

## color_loss 실험 결과 분석

### 브랜치별 Loss 구성 및 결과

| 브랜치 | Loss 구성 | 결과 |
|---|---|---|
| main | L_flow + L_bg + L_lpips + L_edge | 형태 OK, stroke 색 반영 안 됨 |
| exp/color-loss | + L_color (1.5, sketch_latent MSE) | sketch가 그대로 드러남 (악화) |
| exp/color-loss (수정) | - L_color + L_stroke_color (1.0) | 재학습 예정 |

### L_color 실패 원인

`L_color = MSE(pred_latent, sketch_latent)` → sketch_latent는 선화 이미지의 VAE latent.
latent space에서 선의 구조 패턴이 dominant → pred가 sketch처럼 보이게 됨.
색 정보가 없는 선화를 따라가게 강제한 것이 문제.


- `StrokeColorSampler` (augmentation): sketch stroke 색 = GT 머리색으로 교체 (학습 데이터)
- `L_stroke_color` (loss): pred 헤어 평균 색 = sketch stroke 평균 색 강제 (학습 신호)
- 두 개가 맞물려 동작: augmentation이 정답 색을 제공, loss가 그 색으로 학습 유도

### hair-dit와의 비교

hair-dit은 `StrokeColorSampler + L_flow` 조합만으로 색이 나왔음.
hiar-dit3에서 안 된 이유: `L_color (1.5)` 가 L_flow의 색 학습을 방해했기 때문으로 추정.
