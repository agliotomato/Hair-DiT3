# StrokeColorSampler → hair-dit3에서 안 되는 이유

---

## StrokeColorSampler 역할 (hair-dit 기준)

sketch의 stroke 색을 target 이미지의 실제 헤어 픽셀 색으로 교체하는 augmentation.

```
sketch stroke 색 (임의) → target 헤어 픽셀에서 샘플링 → sketch 색 교체

결과: (sketch 색 ≈ 헤어 색, target 헤어 색) 학습 쌍 보장
      → 모델이 "이 색 stroke → 이 색 헤어" correspondence 학습
```

- 33% 확률: target 랜덤 1픽셀 색
- 67% 확률: target 영역 평균 색
- p=1.0: 매 iteration 반드시 적용

hair-dit에서는 p=1.0으로 작동하며 sketch 색 following 확인됨.

---

## hair-dit3에서 p=1.0으로 해도 안 되는 이유

### ctrl_cond 구성은 동일

```python
# hair-dit, hair-dit3 동일
ctrl_cond = cat([sketch_latent + matte_feat, matte_latent], dim=1)  # (B, 17, 64, 64)
```

sketch → VAE encode → sketch_latent 경로도 동일. 색 정보는 latent에 실려 있음.

### 결정적 차이: Matte-Gated Blending

**hair-dit — 없음**

```python
# ControlNet residuals → 모든 token에 그대로 전달
v_pred = transformer(noisy_latents, block_samples=block_samples)
```

**hair-dit3 — 있음**

```python
# 후반 6개 블록: residuals × matte_tokens (0=배경, 1=헤어)
blended[i] = r * matte_tokens
```

### self-attention이 색 전파를 끊는다

SD3.5 MM-DiT의 self-attention은 **1024개 token 전부가 서로를 참조**한다.

**hair-dit (matte-gating 없음)**

```
ControlNet 색 residual → ALL 1024 token에 전달
→ hair token이 attention 시 배경 token도 색 신호를 갖고 있음
→ 색 신호가 attention을 통해 전체로 전파
→ global한 색 컨텍스트 형성 → sketch 색 following 작동
```

**hair-dit3 (matte-gating 있음)**

```
ControlNet 색 residual → hair token에만 전달 (배경 token residual = 0)
→ hair token이 attention 시 배경 token 참조해도 색 신호 없음
→ attention을 통한 색 전파 경로 차단
→ frozen transformer의 hair 색 prior에 밀림 → sketch 색 무시
```

matte-gating은 헤어-배경 경계 품질을 위해 설계됐지만,
**attention의 색 전파 경로를 끊는 부작용**이 있다.

---

## 결론

StrokeColorSampler p=1.0은 "학습 데이터 구성"을 올바르게 만들지만,
hair-dit3 구조에서는 **색 신호가 attention을 통해 전파되지 못하므로** 모델이 이를 활용하지 못한다.

p 값 조정만으로는 해결 불가. 구조적 접근 필요:

| 방향 | 내용 |
|---|---|
| Color Matching Loss | `L_color = \|\|mean_color(output × stroke_mask) - sketch_color\|\|²` |
| 색 conditioning 별도 주입 | per-stroke mean color를 명시적 conditioning으로 추가 |
| matte-gating 범위 조정 | 색 신호를 살리면서 경계 품질 유지하는 blending 비율 재검토 |
